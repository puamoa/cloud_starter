이 실습에서는 AWS를 처음 시작할 때 **가장 먼저** 해야 할 비용 관리 설정을 진행합니다.

AWS는 사용한 만큼 과금되는 종량제 모델입니다. 2025년 7월 15일 이후 신규 가입자는 새로운 크레딧 기반 프리티어가 적용되며, 크레딧이 소진되면 과금이 시작됩니다. 따라서 예산 알림을 먼저 설정하고, 프리티어의 정확한 범위를 이해하는 것이 중요합니다.

> [!WARNING]
> AWS 계정을 생성하면 프리티어가 자동 적용되지만, 크레딧이 소진되거나 유효 기간이 지나면 **즉시 과금**됩니다. 이 실습을 가장 먼저 진행하여 비용 안전장치를 설정하세요.

## 태스크 1: 새 AWS 프리티어 체계 이해 (2025.07.15 이후 신규 가입자)

2025년 7월 15일부터 AWS 프리티어가 크레딧 기반으로 변경되었습니다. 가입 시 **무료 플랜(Free Plan)** 또는 **유료 플랜(Paid Plan)**을 선택합니다.

### 무료 플랜 vs 유료 플랜

| 구분               | 무료 플랜 (Free Plan)                 | 유료 플랜 (Paid Plan) |
| ------------------ | ------------------------------------- | --------------------- |
| **크레딧**         | 가입 시 $100 + 활동 시 추가 최대 $100 | 동일                  |
| **유효 기간**      | 6개월                                 | 6개월                 |
| **서비스 접근**    | 제한된 서비스                         | 150여 개 전체 서비스  |
| **크레딧 소진 후** | 요금 없음 (유료 전환 전까지)          | 사용량에 따라 과금    |
| **적합한 대상**    | 학습, 실험 목적                       | 실제 프로젝트 구축    |

> [!CONCEPT] 크레딧 기반 프리티어
> 기존에는 서비스별로 개별 한도가 있었지만, 새 체계에서는 가입 시 **$100 크레딧**이 기본 제공되고, AWS 서비스를 탐색하며 활동하면 추가로 최대 $100을 더 받을 수 있습니다. 크레딧은 가입일로부터 **6개월간** 유효합니다.
>
> 이론상 발급받은 크레딧으로 이 가이드의 실습을 충분히 진행할 수 있지만, 리소스를 정리하지 않고 방치하면 크레딧이 소진되어 비용이 발생할 수 있습니다.

### 크레딧 소진 주의사항

학습 목적으로 주요 서비스를 사용할 때, 리소스를 정리하지 않고 방치하면 크레딧이 소진될 수 있습니다.

| 서비스          | 비용 수준 | 주의 사항                                                     |
| --------------- | --------- | ------------------------------------------------------------- |
| EC2 t2.micro    | 저렴      | 사용하지 않을 때 Stop하면 EC2 비용 없음                       |
| RDS db.t3.micro | 보통      | 24시간 가동 시 EC2보다 비용 높음, 사용 후 Stop 또는 삭제 권장 |
| S3              | 매우 저렴 | 학습 수준에서는 거의 무시 가능                                |
| NAT Gateway     | 높음      | 시간당 과금으로 크레딧 소진이 빠름, 실습 후 즉시 삭제 필수    |
| EBS 볼륨        | 저렴      | EC2 Stop해도 계속 과금, 하지만 금액은 매우 적음               |

> [!TIP]
> 학습 목적이라면 EC2 + RDS를 동시에 사용해도 크레딧으로 충분히 커버됩니다. 다만 **NAT Gateway**는 비용이 높으므로 실습 후 반드시 삭제하세요. 사용하지 않는 리소스를 주기적으로 정리하는 습관이 중요합니다.

### "항상 무료(Always Free)" 서비스

크레딧과 무관하게 계정이 존재하는 한 영구적으로 무료인 서비스들입니다:

| 서비스     | 항상 무료 한도                    |
| ---------- | --------------------------------- |
| Lambda     | 월 100만 요청 + 40만 GB-초        |
| DynamoDB   | 25GB 저장, 월 2억 5천만 읽기/쓰기 |
| CloudWatch | 기본 모니터링, 알람 10개          |
| SNS        | 월 100만 게시                     |
| SQS        | 월 100만 요청                     |

### 원래 무료인 리소스 (프리티어와 무관)

다음 리소스들은 프리티어 여부와 관계없이 항상 무료입니다:

- **VPC** (Virtual Private Cloud)
- **Subnet** (서브넷)
- **Internet Gateway** (IGW)
- **Route Table** (라우팅 테이블)
- **Security Group** (보안 그룹)
- **Network ACL** (NACL)
- **Parameter Group** (RDS 파라미터 그룹)

> [!NOTE]
> **기존 계정 (2025.07.15 이전 가입자)**: 기존 "12개월 무료 + 서비스별 한도" 체계가 그대로 적용됩니다.

> [!WARNING]
> **크레딧 소진 주의 서비스:**
>
> - **NAT Gateway**: 시간당 과금으로 크레딧이 빠르게 소진됩니다
> - **Elastic IP**: EC2에 연결하지 않은 상태로 방치 시 과금
> - **EBS 볼륨**: EC2를 Stop해도 연결된 EBS는 계속 과금 (금액은 적음)
> - **RDS Multi-AZ**: Single-AZ 대비 2배 비용

✅ **태스크 완료**: 새 AWS 프리티어 체계(무료 플랜 vs 유료 플랜, 크레딧 기반, 6개월)를 이해했습니다.

## 태스크 2: AWS Budgets 예산 알림 설정

이 태스크에서는 월 예산을 설정하고, 임계값에 도달하면 이메일 알림을 받도록 구성합니다.

이 실습을 시작하기 전에 AWS 콘솔 우측 상단에서 리전이 **US East (N. Virginia) us-east-1**로 설정되어 있는지 확인합니다.

> [!NOTE]
> Billing 관련 서비스는 **us-east-1 (버지니아)** 리전에서만 접근할 수 있습니다. 다른 리전에서는 Billing 메뉴가 보이지 않을 수 있습니다.

### 상세 단계

1. AWS Management Console에 로그인합니다.
2. 상단 검색창에 `Billing`을 입력하고 **Billing and Cost Management**를 선택합니다.

> [!NOTE]
> IAM 사용자로 로그인한 경우 Billing 접근 권한이 없을 수 있습니다. Root 계정으로 로그인하거나, Root 계정에서 IAM 사용자의 Billing 접근을 활성화해야 합니다.
>
> **활성화 방법**: Root 계정 로그인 → Account → IAM User and Role Access to Billing Information → Edit → Activate IAM Access → Update

3. 왼쪽 메뉴에서 **Budgets**를 선택합니다.
4. [[Create a budget]] 버튼을 클릭합니다.

> [!NOTE]
> AWS Budgets는 월 2개까지 무료로 생성할 수 있습니다. 3개부터는 예산당 월 $0.02가 부과됩니다.

5. **Budget setup** 섹션에서 다음을 설정합니다:
   - **Budget type**: `Customize (advanced)` 선택
   - **Budget type**: `Cost budget - Recommended` 선택

> [!TIP]
> `Use a template (simplified)`를 선택하면 더 간단하게 설정할 수 있지만, 여기서는 각 옵션을 이해하기 위해 Customize를 선택합니다.

6. [[Next]] 버튼을 클릭합니다.
7. **Set budget amount** 섹션에서 다음을 설정합니다:
   - **Budget name**: `Monthly-Cost-Budget`
   - **Period**: `Monthly`
   - **Budget renewal type**: `Recurring budget`
   - **Start month**: 현재 월 선택
   - **Budgeting method**: `Fixed`
   - **Enter your budgeted amount ($)**: `10.00`

> [!NOTE]
> 학습 목적이라면 $10 정도가 적당합니다. 무료 플랜 크레딧 내에서 사용하면 실제 비용은 $0에 가깝지만, 크레딧 소진이나 실수로 비용이 발생하는 경우를 대비합니다.

8. [[Next]] 버튼을 클릭합니다.
9. **Configure alerts** 섹션에서 [[Add an alert threshold]] 버튼을 클릭합니다.
10. 첫 번째 알림을 설정합니다:
    - **Threshold**: `50` (%)
    - **Trigger**: `Actual` (실제 사용량 기준)
    - **Email recipients**: 본인 이메일 주소 입력

11. [[Add an alert threshold]] 버튼을 다시 클릭하여 두 번째 알림을 추가합니다.
12. 두 번째 알림을 설정합니다:
    - **Threshold**: `80` (%)
    - **Trigger**: `Actual`
    - **Email recipients**: 본인 이메일 주소 입력

13. [[Add an alert threshold]] 버튼을 다시 클릭하여 세 번째 알림을 추가합니다.
14. 세 번째 알림을 설정합니다:
    - **Threshold**: `100` (%)
    - **Trigger**: `Forecasted` (예측 기준)
    - **Email recipients**: 본인 이메일 주소 입력

> [!CONCEPT] Actual vs Forecasted
>
> - **Actual**: 실제 사용 비용이 임계값에 도달하면 알림
> - **Forecasted**: 현재 사용 추세를 기반으로 월말까지 예산을 초과할 것으로 **예측**되면 알림
>
> Forecasted를 100%에 설정하면 아직 실제로 초과하지 않았더라도 미리 경고를 받을 수 있습니다.

15. [[Next]] 버튼을 클릭합니다.
16. **Attach actions** 페이지에서 별도 액션 없이 [[Next]] 버튼을 클릭합니다.

> [!NOTE]
> Budget Actions를 설정하면 예산 초과 시 자동으로 EC2 인스턴스를 중지하거나 IAM 정책을 적용할 수 있습니다. 학습 단계에서는 이메일 알림만으로 충분합니다.

17. **Review** 페이지에서 설정 내용을 확인합니다:
    - Budget name: `Monthly-Cost-Budget`
    - Budget amount: $10.00/month
    - Alert 1: 50% Actual
    - Alert 2: 80% Actual
    - Alert 3: 100% Forecasted

18. [[Create budget]] 버튼을 클릭합니다.

> [!NOTE]
> Budget이 생성되면 Budgets 목록 페이지로 이동합니다. 생성된 Budget을 클릭하면 현재 사용량과 예산 대비 비율을 확인할 수 있습니다.

19. 생성된 `Monthly-Cost-Budget`을 클릭하여 상세 페이지를 확인합니다.
20. **Current vs. budgeted** 그래프에서 현재 사용량이 표시되는지 확인합니다.

> [!TIP]
> 계정을 방금 생성했다면 사용량이 $0.00으로 표시됩니다. 이후 실습을 진행하면서 이 페이지를 주기적으로 확인하세요.

✅ **태스크 완료**: 월 $10 예산 알림이 설정되었습니다. 50%, 80% 도달 시 이메일 알림을 받게 됩니다.

## 태스크 3: Cost Explorer 활성화 및 확인

이 태스크에서는 Cost Explorer를 활성화하여 서비스별, 일별 비용을 시각적으로 확인하는 방법을 학습합니다.

21. Billing and Cost Management 페이지에서 왼쪽 메뉴의 **Cost Explorer**를 선택합니다.

> [!NOTE]
> Cost Explorer를 처음 사용하는 경우 활성화 과정이 필요합니다. 활성화 후 데이터가 표시되기까지 최대 24시간이 소요될 수 있습니다.

22. Cost Explorer 대시보드가 표시되면 다음 항목을 확인합니다:
    - **이번 달 비용**: 현재까지 발생한 총 비용
    - **지난 달 비용**: 전월 총 비용
    - **예측 비용**: 이번 달 예상 총 비용

23. **Group by** 드롭다운에서 `Service`를 선택합니다.
24. 서비스별 비용 분포를 확인합니다.

> [!TIP]
> Cost Explorer에서 자주 확인할 항목:
>
> - **일별 비용 추이**: 갑자기 비용이 증가한 날이 있는지 확인
> - **서비스별 비용**: 어떤 서비스가 가장 많은 비용을 차지하는지 확인
> - **리전별 비용**: 의도하지 않은 리전에서 리소스가 실행되고 있는지 확인
> - **크레딧 잔액**: 남은 크레딧과 소진 속도 확인

25. 왼쪽 메뉴에서 **Free Tier**를 선택합니다.
26. 프리티어 사용량 현황을 확인합니다.

> [!NOTE]
> Free Tier 페이지에서는 각 서비스별로 크레딧 사용량과 항상 무료 서비스의 한도 대비 현재 사용량을 확인할 수 있습니다.

✅ **태스크 완료**: Cost Explorer를 활성화하고 비용 모니터링 방법을 학습했습니다.

## 태스크 4: 추가 비용 방지 설정

이 태스크에서는 크레딧 소진과 비용 폭탄을 방지하기 위한 추가 설정을 진행합니다.

### 태스크 4.1: 프리티어 사용량 알림 활성화

27. Billing and Cost Management → **Billing preferences**를 선택합니다.
28. **Free Tier usage alerts** 섹션에서 **Receive Free Tier usage alerts**를 활성화합니다.
29. 이메일 주소를 입력합니다.
30. [[Save preferences]] 버튼을 클릭합니다.

> [!NOTE]
> 이 설정을 활성화하면 프리티어 한도의 85%에 도달할 때 자동으로 이메일 알림을 받습니다. Budget 알림과 별개로 동작합니다.

### 태스크 4.2: CloudWatch 비용 알람 (선택)

> [!TIP]
> Budget 알림만으로도 충분하지만, 더 세밀한 제어가 필요하다면 CloudWatch 알람을 추가로 설정할 수 있습니다. 이 단계는 선택 사항입니다.

31. 상단 검색창에 `CloudWatch`를 입력하고 선택합니다.
32. 리전을 **US East (N. Virginia)**로 변경합니다.
33. 왼쪽 메뉴에서 **Alarms** → **All alarms**를 선택합니다.
34. [[Create alarm]] 버튼을 클릭합니다.
35. [[Select metric]] 버튼을 클릭합니다.
36. **Billing** → **Total Estimated Charge** → **USD** 선택합니다.
37. **Period**: `6 hours`
38. **Threshold**: `Static`, `Greater than`, `5` (USD)
39. 알림 수신 이메일을 설정합니다.
40. Alarm name: `Billing-Over-5USD`
41. [[Create alarm]] 버튼을 클릭합니다.

> [!NOTE]
> CloudWatch Billing 알람은 us-east-1 리전에서만 생성할 수 있습니다. 6시간마다 체크하여 총 예상 비용이 $5를 초과하면 알림을 보냅니다.

✅ **태스크 완료**: 추가 비용 방지 설정이 완료되었습니다.

# 🗑️ 리소스 정리

> [!NOTE]
> 이 실습에서 생성한 리소스는 모두 무료이므로 삭제하지 않아도 비용이 발생하지 않습니다.

### 단계 1: 리소스 유지 안내

이 실습에서 생성한 리소스는 이후 모든 실습에서 비용 안전장치로 동작하므로 삭제하지 않습니다.

| 리소스                             | 비용              | 유지 이유                      |
| ---------------------------------- | ----------------- | ------------------------------ |
| AWS Budget (`Monthly-Cost-Budget`) | 무료 (월 2개까지) | 비용 초과 시 이메일 알림       |
| Cost Explorer                      | 무료              | 서비스별/일별 비용 모니터링    |
| Free Tier 사용량 알림              | 무료              | 프리티어 한도 85% 도달 시 알림 |
| CloudWatch 알람 (선택 생성한 경우) | 무료 (10개까지)   | 실시간 비용 초과 감지          |

> [!NOTE]
> 이 설정들은 크레딧 소진과 비용 폭탄을 방지하는 핵심 안전장치입니다. 모든 실습이 끝날 때까지 유지하세요.

✅ **실습 종료**: 모든 리소스가 무료이며, 비용 모니터링을 위해 유지됩니다.

## 📚 참고: 추가 $100 크레딧 획득 방법

가입 시 기본 $100 크레딧 외에, 다음 5가지 활동을 완료하면 각 $20씩 추가로 최대 $100을 더 받을 수 있습니다. AWS Management Console의 **Explore AWS** 위젯에서 진행 상황과 크레딧 적립 현황을 확인할 수 있습니다.

### 활동 목록 (각 $20 크레딧)

| #   | 활동                    | 설명                                   | 이 가이드 관련 Step    |
| --- | ----------------------- | -------------------------------------- | ---------------------- |
| 1   | **Amazon EC2** 사용     | EC2 인스턴스를 시작하고 사용해보기     | Step 2 (EC2 서버 구축) |
| 2   | **Amazon RDS** 사용     | RDS 데이터베이스 인스턴스 생성 및 사용 | Step 4 (RDS 관리형 DB) |
| 3   | **AWS Lambda** 사용     | Lambda 함수를 생성하고 실행해보기      | Step 10 (서버리스 백엔드)  |
| 4   | **Amazon Bedrock** 사용 | Bedrock에서 AI 모델 사용해보기         | Step 11 (Bedrock 체험) |
| 5   | **AWS Budgets** 설정    | 예산 알림을 설정하기                   | Step 0 (이 실습)       |

> [!TIP]
> 이 가이드의 실습을 순서대로 진행하면 5개 활동(EC2, RDS, Lambda, Bedrock, Budgets)을 모두 완료하게 됩니다. 추가 $100 크레딧 전액을 받을 수 있습니다.

### 크레딧 적립 확인 방법

1. AWS Management Console에 로그인합니다.
2. 콘솔 홈 화면에서 **Explore AWS** 위젯을 찾습니다.
3. 각 활동의 완료 여부와 적립된 크레딧을 확인합니다.

> [!NOTE]
> 활동 완료 후 크레딧이 적립되기까지 시간이 걸릴 수 있습니다. 적립된 크레딧은 Billing → Credits 페이지에서도 확인할 수 있습니다.
